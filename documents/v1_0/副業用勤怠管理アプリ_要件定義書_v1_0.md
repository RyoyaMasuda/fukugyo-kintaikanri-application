# 副業用勤怠管理アプリ 要件定義書 (Ver 1.0)

## 1. プロジェクト概要
* **アプリ名:** （未定）
* **目的:** 開発者自身のエンジニアリング学習、および友人との実運用
* **ターゲットユーザー:** 開発者本人、およびその友人
* **対応デバイス:** PC（Webブラウザ）

## 2. 機能要件（Functional Requirements）

### 2.1 ユーザー管理機能
* **ユーザー登録:** メールアドレスとパスワードを使用して新規アカウントを作成できる。
* **ログイン:** 登録済みのアカウント情報でシステムにアクセスできる。
* **ログアウト:** 安全にシステムから切断できる。
* **データ分離:** ユーザーは自分自身の勤怠データのみを閲覧・操作できる（他人のデータは見えない）。

### 2.2 勤怠記録機能
* **業務開始（出勤）:** 「開始」ボタンを押下することで、現在時刻を記録できる。
* **業務終了（退勤）:** 「終了」ボタンを押下することで、現在時刻を記録できる。
    * *Ver 1.0 範囲外:* 休憩時間の記録、案件（プロジェクト）の選択、記録後の手動修正。

### 2.3 データ閲覧・集計機能
* **勤務ログ表示:** 自分の過去の勤務記録（日付、開始時間、終了時間）をリスト形式で確認できる。
* **稼働時間集計:** 今月の合計稼働時間を自動計算して表示する。

## 3. 非機能要件（Non-Functional Requirements）
* **ユーザビリティ:** 初心者でも迷わず操作できるシンプルな画面設計（UI）。
* **拡張性:** 将来的に「手動修正機能」や「案件管理機能」を追加しやすい設計にする。
